# Анализ поведения пользователей в соцсети «Сетка»

## Краткое описание  
Проведен анализ пользовательских сессий в социальной сети «Сетка» за январь 2025 года с целью выявления ключевых метрик активности и путей повышения вовлеченности.

## Период анализа  
Январь 2025 года

## Роль и инструменты  
- **Роль:** Data Analyst
- **Инструменты:** PostgreSQL, Python (pandas, numpy, sqlalchemy), Metabase

## Данные
- Таблица `setka_actions` — действия пользователей: просмотр постов,просмотр профилей, лайки, комментарии, подписки, сообщения, репосты
- Основные поля: `session_id`, `user_id`, `action_name`, `action_time`

## Что сделано
- Проведена очистка и подготовка данных (типизация, исправление некорректных значений в столбце action_name)
- Рассчитаны ключевые метрики:
    - Среднее количество действий за сессию — 12
    - Средняя длительность сессии — 50 минут
    - DAU по дням
-Рассчитана конверсия пользователей из просмотра поста в целевые действия:
    - В подписку — 55%
    - В лайк — 62%
    - В просмотр профиля — 61%
    - В сообщение — 55%
    - В комментарий — 60%
    - В репост — 40%
- Построен дашборд в Metabase:
    - Ключевые метрики активности
    - Распределение действий по типам
    - Конверсионная воронка
- Проведен анализ аномалий данных (31 января 2025 года)

## Результат
- Идентифицирован основной сценарий поведения: просмотр поста → просмотр профиля → лайк → комментарий → подписка
- Определены узкие места в продукте: функции «сообщение» и «поделиться» используются редко, возможны проблемы UX
- Подготовлены продуктовые рекомендации для повышения вовлеченности

## Выводы и рекомендации
- Упростить переход от просмотра поста к подписке (CTA-кнопки, подсказки)
- Проработать вовлекающий сценарий: «от лайка к подписке»
- Проверить интерфейс функций «сообщения» и «поделиться»
- Продолжить анализ на последующие месяцы для проверки устойчивости метрик
